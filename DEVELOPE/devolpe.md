# WP WebDav Media Library — Документация разработчика

Этот документ содержит архитектурные стандарты, инструкции по настройке окружения и дорожную карту (Roadmap) для независимого развития плагина.

Плагин является самостоятельным форком проекта [external-files-from-webdav](https://github.com/threadi/external-files-from-webdav).

## 1. Базовые константы и стандарты (Ребрендинг)

Чтобы избежать конфликтов с другими плагинами (включая оригинальный), при написании нового кода и рефакторинге старого строго используйте следующие стандарты:

- **Text Domain:** `wp-webdav-media-library` (используется для всех функций перевода `\_\_()`, `\_e()`).

- **Префикс функций и переменных:** `wwml\_` (сокращение от WP WebDav Media Library).

- **Глобальные константы:** Начинаются с `WWML\_` (например, `WWML\_PLUGIN\_DIR`).

- **Пространства имен (Namespaces):** Если внедряется ООП-подход, используйте KiSa`\\WebDavMediaLibrary`.


## 2. Дорожная карта (Roadmap) на ближайшие релизы

### Шаг 1: Полное отсоединение от оригинала

- [ ] Переименовать главную папку и `.php` файл в `wp-webdav-media-library`.

- [ ] Заменить все старые префиксы `external\_files\_` и `threadi\_` на `wwml\_`.

- [ ] Обновить Plugin Header (название - WP WebDav Media Library, автор - KiSa, описание — пока аналогичное).

- [ ] Заменить text-domain во всех файлах.

- [ ] Переписать `README.md` для нового репозитория.

### Шаг 2: Новый функционал (Запланировано)

- [ ] Улучшить UI страницы настроек WebDAV в админке.

- [ ] Добавить поддержку новых провайдеров (Nextcloud, Яндекс.Диск и т.д.).

- [ ] Оптимизировать кэширование превью изображений.

## 4. Правила коммитов (Git Workflow)

Все новые функции сливаются в `main`. Коммиты на русском языке. Формат сообщений коммитов:

- `feat:` — новый функционал

- `fix:` — исправление ошибки

- `refactor:` — переписывание кода без изменения функционала

- `docs:` — обновление документации
